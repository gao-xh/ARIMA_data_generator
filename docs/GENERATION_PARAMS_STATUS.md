## 2. Missing Initialization Values (To Decisions)

While we have the static metadata, the **dynamic behavioral parameters** (how the drug *moves*) are currently using generic defaults. To increase realism, we might need:

### 2.1 Daily Volume (Throughput)
*   **Current State**: Randomly generated by `DemandModel` (e.g. 10-50 units).
*   **Missing**: A specific "Velocity" for each SKU.
    *   *Real World*: Is "Dexamethasone" a fast mover (100/day) or slow mover (1/day)?
    *   *Decision*: Do we add a column `日均销量预估` to `drug_info.csv`? Or infer it from `波动区间分类` (e.g., Low Vol params usually = High Volume?)?

### 2.2 Seasonality Profile (The "Peak")
*   **Current State**: All drugs share the same Global Flu Season (Dec-Feb).
*   **Missing**: Drug-specific seasonality.
    *   *Real World*: Antibiotics peak in winter. Stomach medicine might peak in summer (food poisoning)? Chronic drugs (Diabetes) represent flat demand.
    *   *Decision*: Currently treated as global. For thesis rigor, we might need to map `药品品类` to a seasonality curve type.
    
### 2.3 Starting Inventory (Day 0)
*   **Current State**: Tuner assumes "Steady State" (2 months supply).
*   **Missing**: Specific starting backlog.
    *   *Impact*: Low. The first 30 days of simulation ("Burn-in period") usually wash out the initial state effects.
